# 我的中餐厅 v1.4 需求文档

**版本：** v1.4 全面修复版
**日期：** 2026-03-29
**状态：** 待开发

---

## 📋 问题汇总（v1.3 未修复）

### 问题 1：厨房 Tab 员工分类子 Tab 点击无效
**现象：** 能看到"厨师/服务员/收银员"三个子 Tab，但点击没有任何反应
**影响：** 无法切换查看不同员工类型
**优先级：** 🔴 高

### 问题 2：首页收银员消失
**现象：** 首页看不到收银员
**影响：** 首页视觉不完整
**优先级：** 🟡 中

### 问题 3：经营 Tab 子导航点击区域不准
**现象：** 
- 必须点击"厨房"和"采购"按钮的上方才有效
- 点击按钮本身无效
**影响：** 用户体验差，操作困难
**优先级：** 🔴 高

### 问题 4：顾客走动颤动（待确认）
**现象：** 顾客在小区域内颤动，不是流畅走动
**影响：** 视觉效果差
**优先级：** 🟡 中

---

## 🎯 核心需求

### 需求 1：所有 Tab 和子 Tab 必须可点击

**功能描述：**
- 底部导航：首页、经营、社交、商城 - 4 个 Tab 必须可点击切换
- 经营子导航：餐厅、厨房、采购、外卖 - 4 个子 Tab 必须可点击切换
- 厨房员工分类：厨师、服务员、收银员 - 3 个子 Tab 必须可点击切换

**验收标准：**
1. 点击任意 Tab/子 Tab，能立即切换
2. 点击后对应区域高亮（红色背景）
3. 点击后内容区域正确显示
4. 点击成功率 100%

**技术实现：**
- 检查触摸事件检测范围是否与渲染位置匹配
- 检查点击坐标计算是否正确
- 检查事件处理函数是否正确调用

---

### 需求 2：首页必须显示完整场景

**功能描述：**
首页餐厅场景应包含：
- 地板区域（顾客走动区域）
- 桌椅家具（可拖拽）
- 收银台（右侧）
- 收银员（在收银台位置）
- 厨师（在厨房区域）
- 顾客（自动进店走动）

**验收标准：**
1. 收银员在收银台位置可见
2. 厨师在厨房位置可见
3. 顾客在地板区域走动
4. 家具可拖拽摆放

**技术实现：**
- 检查首页渲染函数是否正确绘制所有角色
- 检查角色坐标是否在可见区域内
- 检查角色动画是否正常启动

---

### 需求 3：顾客走动流畅自然

**功能描述：**
- 顾客从右侧门口进入
- 走到座位区域
- 等待上菜→吃饭→付款→离开
- 整个过程流畅，不颤动

**验收标准：**
1. 顾客移动速度适中（不慢不卡）
2. 移动轨迹平滑（不颤抖）
3. 遇到家具会绕行
4. 完成完整流程后离开

**技术实现：**
- 移动速度 ≥ 3 像素/帧
- 更新频率 ≥ 20fps
- 避障逻辑正确

---

### 需求 4：厨房 Tab 功能完整

**功能描述：**
厨房 Tab 包含：
- 员工分类子 Tab（厨师/服务员/收银员）
- 员工列表（显示当前分类的员工）
- 雇佣员工按钮
- 成本统计卡片

**验收标准：**
1. 三个员工分类子 Tab 显示完整文字
2. 点击子 Tab 能切换员工列表
3. 雇佣按钮可点击
4. 成本统计正确显示

**技术实现：**
- 修复子 Tab 点击检测范围
- 确保员工数据正确存储和读取
- 雇佣逻辑正确实现
