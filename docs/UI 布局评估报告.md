### 1.2 核心问题量化分析

#### 问题 1：小屏幕拥挤风险

| 屏幕宽度 | 单 Tab 可用宽度 | 图标 + 文字需求 | 状态 |
|---------|--------------|---------------|------|
| 320px (iPhone SE) | 64px | 图标 24px + 文字 40px | ⚠️ **临界** |
| 375px (iPhone 8) | 75px | 图标 24px + 文字 40px | ⚠️ **紧张** |
| 414px (iPhone 11) | 82px | 图标 24px + 文字 40px | ✅ 可接受 |
| 428px (iPhone 14 Pro Max) | 85px | 图标 24px + 文字 40px | ✅ 舒适 |

**风险点**：
- 微信小游戏**最低支持 320px** 宽度
- 5 Tab 在 320px 屏幕上，单 Tab 仅 64px
- 图标 24px + 文字间距 4px + 文字约 36px = **刚好塞满，无呼吸空间**

#### 问题 2：点击区域不足

```
iOS 人机界面指南要求：
- 最小点击区域：44×44pt
- 推荐点击区域：48×48pt

当前 5 Tab 布局（320px 屏幕）：
┌─────────────────────────────────────────────────────────┐
│  [64px] [64px] [64px] [64px] [64px]                     │
│    ↑      ↑      ↑      ↑      ↑                        │
│  勉强够  勉强够  勉强够  勉强够  勉强够                   │
└─────────────────────────────────────────────────────────┘

安全点击区域需要：
┌─────────────────────────────────────────────────────────┐
│  [≥44px][≥44px][≥44px][≥44px][≥44px] = 220px 最小       │
│  + 间距 4px×4 = 16px                                    │
│  + 左右边距 16px×2 = 32px                               │
│  = 268px 最小需求                                       │
│                                                         │
│  实际可用 320px - 268px = 52px 余量                      │
│  52px ÷ 5 = 10.4px/Tab 余量（危险）                      │
└─────────────────────────────────────────────────────────┘
```

#### 问题 3：认知负荷偏高

**米勒定律**：人类短期记忆容量为 7±2 个信息块

| Tab 数量 | 认知负荷 | 用户记忆难度 |
|---------|---------|------------|
| 3 Tab | 低 | 轻松记忆 |
| 4 Tab | 中低 | 较轻松 |
| **5 Tab** | **中高** | **需要思考** |
| 6 Tab+ | 高 | 容易混淆 |

**5 Tab 的问题**：
- 餐厅 vs 厨房：功能边界模糊（都是经营相关）
- 外卖：是独立功能还是厨房子功能？
- 用户需要**5 选 1**决策，增加思考时间

#### 问题 4：扩展性已达上限

```
当前：[餐厅][厨房][外卖][社交][商城] = 5/5 (100% 占用)

未来可能新增功能：
- 活动/任务中心？→ 无位置
- 排行榜？→ 无位置
- 设置/个人信息？→ 无位置
- 邮件/消息？→ 无位置

结果：必须重构或放弃新功能
```

### 3.2 方案 A：保持 5 Tab（不推荐）

```
[🏠 餐厅] [🍳 厨房] [🛵 外卖] [📱 社交] [💎 商城]
```

**优点**：
- ✅ 功能入口直接，无需二次点击
- ✅ 用户学习成本低（所见即所得）
- ✅ 开发成本最低（无需改动）

**缺点**：
- ❌ 小屏幕（320px）显示拥挤
- ❌ 点击区域处于临界值（44px）
- ❌ 无扩展空间（已达上限）
- ❌ 餐厅/厨房功能边界模糊

**改进建议**（如坚持此方案）：
```css
/* 必须实现的优化 */
.tab-button {
  min-width: 64px;      /* 最小宽度保证 */
  min-height: 56px;     /* 点击区域≥44px */
  padding: 8px 4px;     /* 内边距增加点击区 */
  font-size: 10px;      /* 文字缩小适配 */
}

.tab-icon {
  width: 24px;
  height: 24px;
  margin-bottom: 4px;
}

/* 320px 屏幕特殊处理 */
@media (max-width: 320px) {
  .tab-text {
    display: none;      /* 仅显示图标 */
  }
}
```

**风险评估**：🔴 **高风险** - 可能导致 15-20% 小屏幕用户点击体验差

### 3.3 方案 B：合并功能（强烈推荐）⭐

```
[🏠 首页] [🍳 经营] [📱 社交] [💎 商城]
              ↓
         [餐厅/厨房/外卖]
```

**合并逻辑**：
```
经营 Tab（核心玩法集合）：
├─ 餐厅（主场景）
├─ 厨房（制作区）
└─ 外卖（订单流）

用户进入经营 Tab 后，通过子导航或场景切换访问具体功能
```

**优点**：
- ✅ 符合 4 Tab 主流设计（65% 竞品选择）
- ✅ 小屏幕适配安全（80px/Tab）
- ✅ 点击区域充足（56×56px）
- ✅ 保留 1 个扩展位
- ✅ 餐厅/厨房逻辑合并（都是经营场景）

**缺点**：
- ⚠️ 外卖入口深度 +1（需二次点击）
- ⚠️ 需要设计经营 Tab 内导航
- ⚠️ 用户需要重新学习（短期适应成本）

**实现方案**：
```
经营 Tab 内部设计：
┌─────────────────────────────────────────────────────────┐
│  [餐厅] [厨房] [外卖]  ← 顶部子导航                      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│                    当前场景内容                          │
│                                                         │
└─────────────────────────────────────────────────────────┘
│  [首页] [经营▼] [社交] [商城]  ← 底部主导航               │
└─────────────────────────────────────────────────────────┘
```

**视觉设计建议**：
```css
经营 Tab 激活状态：
- 底部 Tab 高亮 + 向下箭头指示
- 顶部子导航显示当前子功能
- 场景平滑切换（fade 或 slide）

子导航样式：
- 高度：40px
- 激活态：下划线 2px 中国红
- 文字：12px，深棕色
```

**扩展性**：
```
当前：[首页][经营][社交][商城] = 4/5 (80% 占用)

未来可新增：
- [活动] Tab（限时活动时显示）
- [排行榜] Tab（社交深化）
- [邮件] Tab（消息通知）

或保持 4 Tab，在经营 Tab 内扩展子功能
```

**风险评估**：🟢 **低风险** - 符合行业最佳实践

### 3.4 方案 C：二级导航（推荐）

```
[🏠 首页] [🍳 经营▼] [📱 社交] [💎 商城]
              ↓
         弹出：[餐厅][厨房][外卖]
```

**交互逻辑**：
```
点击"经营▼" → 弹出气泡菜单 → 选择子功能

气泡菜单设计：
┌─────────────────┐
│  [🏠 餐厅]      │
│  [🍳 厨房]      │
│  [🛵 外卖]      │
└─────────────────┘
```

**优点**：
- ✅ 底部保持 4 Tab，小屏幕安全
- ✅ 子功能入口清晰（一级展示）
- ✅ 扩展性强（子菜单可容纳 5+ 项）
- ✅ 视觉层次分明

**缺点**：
- ⚠️ 外卖入口深度 +1（二次点击）
- ⚠️ 弹出菜单增加交互步骤
- ⚠️ 需要处理菜单遮挡问题

**实现方案**：
```javascript
// 伪代码示例
经营 Tab 点击处理：
onClick() {
  if (子菜单已打开) {
    关闭子菜单;
  } else {
    显示子菜单 (位置：经营 Tab 正上方);
    高亮经营 Tab;
  }
}

子菜单选择处理：
onSelect(子功能) {
  切换到对应场景;
  关闭子菜单;
  保持经营 Tab 高亮;
}
```

**视觉设计**：
```css
弹出菜单样式：
- 背景：白色卡片，圆角 8px
- 阴影：0 4px 12px rgba(0,0,0,0.15)
- 每项高度：48px
- 图标：24×24px
- 文字：14px，深棕色
- 选中态：浅红色背景 #FFF0F0

动画：
- 出现：fade + slide up，0.2s
- 消失：fade + slide down，0.15s
```

**风险评估**：🟡 **中低风险** - 需要良好交互设计
